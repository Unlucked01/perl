# Спецификация требований к проекту "Матрица" на Perl

## Общее описание
Проект представляет собой веб-приложение на основе CGI-Perl для работы с матрицами. Приложение позволяет пользователям создавать матрицы и выполнять различные операции над ними через веб-интерфейс на русском языке.

## Функциональные требования

### Класс "Матрица"
Основной модуль должен реализовать класс `Matrix` со следующими возможностями:
- Создание матрицы заданных размеров
- Вывод матрицы в понятном формате
- Транспонирование матрицы
- Определение максимального и минимального элементов
- Сложение с другой матрицей
- Умножение на другую матрицу
- Вычисление обратной матрицы

### Веб-интерфейс
- Форма для создания матриц различных размеров
- Возможность заполнения матрицы вручную или случайными числами
- Интерфейс для выбора операции над матрицей
- Вывод результатов выполнения операций
- Весь интерфейс на русском языке

## Структура проекта

```
/matrix/
├── Dockerfile
├── httpd.conf
├── start.sh
├── cgi-bin/
│   ├── matrix.pl        # Основной скрипт CGI
│   └── Matrix.pm        # Модуль с классом Matrix
└── public_html/
    ├── index.html       # Главная страница
    ├── style.css        # Стили для веб-интерфейса
    └── script.js        # JavaScript для веб-интерфейса
```

## Инфраструктура

### Dockerfile
Будет основан на образе httpd:2.4, как в примере spellcheck, с установкой необходимых Perl модулей.

### httpd.conf
Конфигурация Apache будет аналогична примеру spellcheck, с настройкой порта 8082 для матричного сервиса.

### start.sh
Скрипт для сборки и запуска Docker-контейнера, аналогичный примеру spellcheck, но с другим названием сервиса и портом.

## Интерфейс пользователя

Веб-интерфейс будет включать:
- Форму для задания размерности матриц
- Поля для ввода элементов матриц
- Кнопки для выбора операций
- Область для отображения результатов

Все элементы интерфейса и сообщения будут на русском языке.

## Технические ограничения
- Ограничение на максимальные размеры матриц (для предотвращения перегрузки сервера)
- Проверка корректности ввода данных перед выполнением операций
- Обработка ошибок при невозможности выполнения операций (например, при несовпадении размерностей)

## План реализации
1. Разработка модуля Matrix.pm
2. Создание CGI-скрипта matrix.pl
3. Разработка HTML/CSS/JS для веб-интерфейса
4. Настройка Docker-инфраструктуры
5. Тестирование и отладка
