

# Проектная документация (PRD): Веб-приложение "Сайт научного журнала"

## 1. Введение

Данный документ описывает требования и план реализации веб-приложения "Сайт научного журнала". Система предназначена для публикации электронных версий научных статей, управления выпусками журнала, обработки заказов пользователей и предоставления статистики по продажам.

## 2. Цели проекта

- Разработать полнофункциональный сайт научного журнала
- Обеспечить доступ к электронным версиям выпусков журнала
- Реализовать систему регистрации и авторизации пользователей
- Создать механизм формирования "корзины" и оформления заказов
- Разработать функционал для приема статей от авторов
- Предоставить инструменты для анализа статистики продаж

## 3. Целевая аудитория

Система предназначена для следующих категорий пользователей:
- **Покупатели** - читатели, заинтересованные в приобретении статей
- **Редакционная коллегия** - сотрудники, ответственные за публикацию статей
- **Администраторы (менеджеры)** - персонал, управляющий заказами и системой в целом

## 4. Технические требования

### 4.1. Платформа и окружение

- **Операционная система**: Windows или Linux (преимущественно Red Hat)
- **Веб-сервер**: Apache с поддержкой CGI
- **Языки программирования**: Perl, JavaScript, HTML, CSS
- **База данных**: DB-файлы (Berkeley DB)

### 4.2. Структура сайта

- Не менее 5 страниц, включая главную страницу
- Каждая страница должна содержать:
  - Текстовый контент
  - Рисунки
  - Фоновый рисунок
  - Таблицы (желательно)
- Навигация внутри страницы:
  - Переход на начало страницы
  - Переход в конец страницы
  - Переходы по тематическим разделам
  - Навигация по списку рисунков
  - Возврат на главную страницу

### 4.3. Динамическая генерация страниц

- Минимум 3 страницы должны генерироваться динамически с помощью CGI-программ на Perl
- Динамически сгенерированные страницы должны содержать:
  - Гиперссылки или кнопки submit для обработки форм
  - Изображения или кнопки, оформленные в виде изображений
  - Кнопки со связанными JavaScript-сценариями

### 4.4. База данных

- Структура: не менее 3 связанных таблиц
- Реализация: DB-файлы
- Доступ: предоставление пользователям возможности просмотра данных

## 5. Функциональные требования

### 5.1. Основные функции

#### 5.1.1. Работа с выпусками журнала
- Отображение электронных версий выпусков по номерам
- Просмотр тезисов и полных текстов статей
- Формирование и публикация новых выпусков

#### 5.1.2. Управление пользователями
- Регистрация новых пользователей
- Авторизация существующих пользователей
- Управление профилями пользователей

#### 5.1.3. Система заказов
- Формирование "корзины" пользователя
- Управление содержимым корзины
- Оформление заказа
- Генерация квитанций для оплаты через почту или банк
- Изменение статуса заказа менеджером

#### 5.1.4. Работа с авторами
- Прием тезисов и статей к публикации
- Управление статусом рассмотрения статей
- Обратная связь с авторами

#### 5.1.5. Аналитика
- Определение статистики продаваемости статей по авторам
- Анализ популярности статей по тематике
- Формирование отчетов для редакционной коллегии

### 5.2. Роли пользователей

#### 5.2.1. Покупатель
- Просмотр выпусков журнала и статей
- Добавление статей в корзину
- Оформление и оплата заказов
- Просмотр истории заказов

#### 5.2.2. Редакционная коллегия
- Все функции покупателя
- Прием и рассмотрение статей от авторов
- Формирование новых выпусков журнала
- Редактирование содержимого сайта

#### 5.2.3. Администратор (менеджер)
- Все функции редакционной коллегии
- Управление пользователями
- Изменение статусов заказов
- Доступ к статистике и аналитике
- Настройка параметров системы

## 6. Структура базы данных

### 6.1. Таблицы

#### 6.1.1. Пользователи (Users)
- ID пользователя (первичный ключ)
- Логин
- Пароль (хешированный)
- ФИО
- Email
- Телефон
- Адрес
- Роль (покупатель, редактор, администратор)
- Дата регистрации

#### 6.1.2. Выпуски журнала (Issues)
- ID выпуска (первичный ключ)
- Номер выпуска
- Год издания
- Месяц издания
- Название выпуска
- Описание
- Обложка (путь к файлу)
- Статус (опубликован, в подготовке)
- Дата публикации

#### 6.1.3. Статьи (Articles)
- ID статьи (первичный ключ)
- ID выпуска (внешний ключ)
- Название статьи
- Авторы
- Тезисы
- Полный текст (путь к файлу)
- Тематика
- Ключевые слова
- Цена
- Дата добавления

#### 6.1.4. Заказы (Orders)
- ID заказа (первичный ключ)
- ID пользователя (внешний ключ)
- Дата создания
- Сумма заказа
- Статус (новый, оплачен, выполнен, отменен)
- Способ оплаты
- Номер квитанции

#### 6.1.5. Детали заказа (OrderDetails)
- ID записи (первичный ключ)
- ID заказа (внешний ключ)
- ID статьи (внешний ключ)
- Количество
- Цена на момент заказа

#### 6.1.6. Авторские заявки (Submissions)
- ID заявки (первичный ключ)
- ID пользователя (внешний ключ)
- Название статьи
- Авторы
- Тезисы
- Файл статьи (путь к файлу)
- Тематика
- Ключевые слова
- Дата подачи
- Статус (на рассмотрении, принята, отклонена, опубликована)
- Комментарии редакции

## 7. Пользовательский интерфейс

### 7.1. Главная страница
- Логотип и название журнала
- Краткая информация о журнале
- Последние выпуски
- Популярные статьи
- Новости и анонсы
- Форма авторизации/регистрации
- Навигационное меню

### 7.2. Страница выпусков
- Список всех выпусков с фильтрацией по году
- Обложки и краткая информация о выпусках
- Возможность перехода к содержанию выпуска

### 7.3. Страница выпуска
- Информация о выпуске
- Содержание (список статей)
- Возможность просмотра тезисов
- Кнопки добавления статей в корзину

### 7.4. Страница статьи
- Название и авторы
- Тезисы
- Возможность просмотра полного текста (для авторизованных пользователей)
- Информация о цене
- Кнопка добавления в корзину

### 7.5. Личный кабинет
- Профиль пользователя
- История заказов
- Доступные статьи
- Для авторов: форма подачи статей и статус рассмотрения

### 7.6. Корзина
- Список выбранных статей
- Управление количеством
- Удаление из корзины
- Расчет общей суммы
- Кнопка оформления заказа

### 7.7. Оформление заказа
- Проверка данных пользователя
- Выбор способа оплаты
- Генерация квитанции
- Подтверждение заказа

### 7.8. Административная панель
- Управление пользователями
- Управление выпусками и статьями
- Обработка заказов
- Рассмотрение авторских заявок
- Статистика и аналитика

## 8. Процессы и сценарии использования

### 8.1. Регистрация и авторизация
1. Пользователь заполняет форму регистрации
2. Система проверяет данные и создает учетную запись
3. Пользователь авторизуется с помощью логина и пароля
4. Система предоставляет доступ к функциям в соответствии с ролью

### 8.2. Просмотр и покупка статей
1. Пользователь просматривает список выпусков
2. Выбирает интересующий выпуск
3. Просматривает список статей и их тезисы
4. Добавляет выбранные статьи в корзину
5. Переходит в корзину и оформляет заказ
6. Получает квитанцию для оплаты
7. После подтверждения оплаты получает доступ к полным текстам

### 8.3. Подача статьи автором
1. Автор авторизуется в системе
2. Заполняет форму подачи статьи
3. Прикрепляет файл с текстом статьи
4. Отправляет заявку на рассмотрение
5. Получает уведомления об изменении статуса заявки

### 8.4. Публикация нового выпуска
1. Редактор формирует новый выпуск
2. Добавляет принятые статьи в выпуск
3. Заполняет информацию о выпуске
4. Публикует выпуск на сайте

### 8.5. Обработка заказов
1. Менеджер просматривает список новых заказов
2. Проверяет факт оплаты
3. Изменяет статус заказа
4. Система предоставляет доступ пользователю к купленным статьям

## 11. Заключение

Веб-приложение "Сайт научного журнала" предоставит полный набор функций для публикации и распространения научных статей, управления выпусками журнала и обработки заказов пользователей. Система будет удовлетворять всем техническим требованиям, включая динамическую генерацию страниц с помощью CGI-программ на Perl и использование DB-файлов для хранения данных.
